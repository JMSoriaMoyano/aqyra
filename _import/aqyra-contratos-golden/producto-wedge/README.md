# producto-wedge/ — Track B · capa BIM/cumplimiento

El producto de entrada al mercado: **Visor OpenBIM + auditoría ISO 19650 + checklist CTE**. Vende **proceso y documentación, no firma estructural** — baja responsabilidad, dolor recurrente, comprador amplio.

## PRD/ (Documento de Requisitos de Producto)

Define **qué** se construye y **por qué**, no el cómo. Contenido objetivo (tareas B1.1–B1.5):

- Problema y **buyer persona** (BIM managers, oficinas de proyecto).
- **Propuesta de valor.**
- **Alcance del MVP** (qué hace) y, sobre todo, la **frontera** (qué NO hace — sin cálculo de firma).
- Criterios de aceptación y métricas de éxito.
- Hipótesis de **pricing** (1.200–2.400 €/empresa/año, orientativo).

## Piloto externo de referencia

**Can Cabassa** (ver `../pilotos/can-cabassa/`) valida el alcance del PRD y es el primer objetivo de **LOI**.
